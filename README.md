# QA Portfolio - Василий Луцек

Привет! Это мое практическое QA-портфолио.
Здесь собраны материалы, с которыми я обычно работаю как junior QA: API-тестирование в Postman, документация по API, SQL-запросы.

Проект сделан как "рабочая папка" с реальными артефактами, а не только как красивый шаблон.

## Что внутри

- `DemoShopping.postman_collection.json` - коллекция Postman для API интернет-магазина.
- `README_DemoShopping.md` - подробная инструкция по запуску и структуре коллекции.
- `openapi.json` - OpenAPI/Swagger спецификация API (22 endpoint-пути, 28 методов).
- `SQL_Select.sql` - набор SQL-запросов для тренировок и проверки данных.
- `portfolio-html/` - простая html/css-страница портфолио (контакты + ссылки на артефакты).
- `qa-interview-questions-60.md` - 60 популярных QA-вопросов с эталонными ответами.
- `stepik-certificate-248274-7b39e18.pdf` - сертификат о прохождении обучения.

## Что я здесь отрабатываю

- написание и запуск API-тестов (позитивных и негативных);
- работу с токенами, переменными и окружениями в Postman;
- проверку контрактов и структуры ответов API;
- базовые проверки данных через SQL;
- аккуратное оформление QA-артефактов под портфолио и отклики.

## Быстрый старт

1. Импортируйте `DemoShopping.postman_collection.json` в Postman.
2. Настройте переменные окружения .
3. Запустите `login` или `register`, чтобы получить токен.
4. Прогоните сценарии из `Products` и `Cart`.
5. Для деталей используйте `README_DemoShopping.md`.

## Коротко про стек

- **Инструменты:** Postman, Swagger/OpenAPI, SQL, Chrome DevTools
- **База:** клиент-серверная архитектура, HTTP, тест-дизайн
