# DemoShopping Postman Collection

Коллекция `DemoShopping.postman_collection.json` предназначена для тестирования API интернет-магазина:
- авторизация и регистрация;
- операции с товарами (поиск, фильтрация, CRUD);
- операции с корзиной пользователя.

## Содержимое коллекции

- **login**: авторизация пользователя и сохранение токена в переменную окружения `key`.
- **register**: регистрация нового пользователя и сохранение токена в `key`.
- **Products**:
  - получение всех товаров;
  - поиск товара по `id`;
  - фильтрация по категории, производителю, доставке и диапазону цены;
  - добавление, полное и частичное обновление, удаление товара.
- **Cart** (требует авторизацию):
  - получить корзину;
  - добавить товар в корзину;
  - обновить количество;
  - удалить товар из корзины.

## Базовые переменные

В коллекции используется:
- `baseUrl` (по умолчанию: `https://intern.demoshopping.ru`);
- `key` (JWT-токен; устанавливается после `login` или `register`);
- `productId` (id товара для запросов по товару и добавления в корзину);
- `cartItemId` (id позиции корзины для обновления/удаления).

## Быстрый старт в Postman

1. Импортируйте `DemoShopping.postman_collection.json` в Postman.
2. Создайте Environment (например, `DemoShopping Local`).
3. Добавьте в Environment переменные:
   - `baseUrl=https://intern.demoshopping.ru`
   - `productId=<существующий id товара>`
   - `cartItemId=<id позиции корзины>` (можно заполнить позже)
4. Выполните `login` (или `register`) и убедитесь, что токен записался в `key`.
5. Запускайте запросы из папок `Products` и `Cart`.

## Рекомендуемый порядок прогона

1. `login` или `register`
2. `Products -> Возвращает список всех продуктов`
3. `Products -> Поиск товара по ID`
4. `Cart -> Добавляет товар в корзину пользователя`
5. `Cart -> Возвращает содержимое корзины пользователя`
6. `Cart -> Обновляет количество товара в корзине`
7. `Cart -> Удаляет товар из корзины`

## Проверки в тест-скриптах

В большинстве запросов реализованы проверки:
- HTTP-статуса (`200`, `400`, `403`, `404`);
- времени ответа (обычно `< 3000 ms`);
- структуры ответа (`array`, обязательные поля, типы данных);
- бизнес-условий (например, `freeShipping=true`, совпадение `manufacturer/category`, корректное обновление полей).

## Запуск через Newman (опционально)

```bash
newman run DemoShopping.postman_collection.json -e DemoShopping.postman_environment.json
```

Если environment-файла нет, сначала экспортируйте его из Postman после настройки переменных.

## Важное замечание по безопасности

В JSON-коллекции может храниться bearer-токен. Перед публикацией в GitHub рекомендуется:
- удалить фиксированные токены из коллекции;
- использовать только переменные окружения;
- не коммитить реальные секреты.
